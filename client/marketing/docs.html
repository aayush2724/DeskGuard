DeskGuard Developer Docs

Everything you need to read live occupancy data, embed the floor map, and connect DeskGuard to your own systems.

Quickstart

Every DeskGuard account includes an API key, available from Dashboard → Settings → API. All requests use this key as a bearer token.

curl https://api.deskguard.io/v1/floors/floor-2b/desks \
  -H "Authorization: Bearer dg_live_xxxxxxxxxxxx"

A successful response returns the current status of every desk on that floor:

{
  "floor": "floor-2b",
  "updated_at": "2026-06-13T10:32:00Z",
  "desks": [
    { "id": "2b-014", "status": "free" },
    { "id": "2b-015", "status": "occupied" },
    { "id": "2b-016", "status": "ghost", "ghost_since": "2026-06-13T10:09:00Z" }
  ]
}

Authentication

API keys are scoped per-account and can be restricted to read-only access. Rotate keys at any time from the dashboard — old keys remain valid for 24 hours after rotation to avoid downtime.

For embedded floor maps (see Embedding), use a separate public embed token, which only exposes aggregate desk status and cannot be used to access account settings or analytics exports.

REST endpoints

MethodEndpointDescription
GET/v1/floorsList all floors on your account
GET/v1/floors/:id/desksCurrent status of every desk on a floor
GET/v1/floors/:id/desks/:desk_id/historyStatus history for a single desk
GET/v1/analytics/utilisationAggregate utilisation stats, filterable by date range and floor
POST/v1/alerts/testSend a test alert to your configured channel

WebSocket feed

For live displays, subscribe to the WebSocket feed instead of polling. Desk status pushes occur roughly every 30 seconds, or immediately when a desk's status changes.

const ws = new WebSocket("wss://api.deskguard.io/v1/stream?token=dg_live_xxxxxxxxxxxx");

ws.onmessage = (event) => {
  const update = JSON.parse(event.data);
  // { floor: "floor-2b", desk: "2b-016", status: "ghost", ghost_since: "..." }
  console.log(update);
};

Desk status values

StatusMeaning
freeNo device or human presence detected
occupiedHuman presence detected at the desk
ghostDevice present, no human presence for longer than the configured ghost-seat window
offlineSensor for this desk is not reporting — check sensor health in the dashboard

Embed the floor map

Add a live, interactive floor map to your own site or intranet with a single script tag. Replace YOUR_EMBED_TOKEN and FLOOR_ID with values from Dashboard → Settings → Embeds.

<div id="deskguard-map"></div>
<script src="https://embed.deskguard.io/v1/map.js"
        data-token="YOUR_EMBED_TOKEN"
        data-floor="FLOOR_ID"
        data-target="#deskguard-map"></script>

The embed is responsive and matches your page's colour scheme automatically by reading prefers-color-scheme, or you can force a theme with data-theme="dark" or data-theme="light".

Kiosk mode

For wall-mounted displays, visit https://app.deskguard.io/kiosk/FLOOR_ID?token=YOUR_EMBED_TOKEN in any browser and set it to fullscreen. Kiosk mode automatically refreshes, dims during off-hours based on your configured schedule, and recovers from network drops without manual intervention.

Booking systems

DeskGuard can overlay reservation data from supported booking systems so the floor map distinguishes "reserved, not yet occupied" from "walk-in available." Supported integrations:

  • LibCal — connect via OAuth in Dashboard → Integrations
  • Outlook / Microsoft 365 — requires admin consent for room-resource calendars
  • Google Calendar — connect a service account with access to resource calendars

Once connected, reserved desks show a small calendar icon on the floor map and in the embed, even before the reservation start time.

Webhooks & alerts

Configure webhooks from Dashboard → Settings → Webhooks to receive POST requests on key events:

EventTrigger
desk.ghostA desk crosses the ghost-seat timeout threshold
desk.offlineA sensor stops reporting for more than 5 minutes
floor.utilisation_thresholdFloor occupancy crosses a configured percentage

Slack and Microsoft Teams have built-in integrations that don't require custom webhook handling — connect them directly from the same settings page.
